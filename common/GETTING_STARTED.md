# Vibe Coding 入门指南

> 从零到一，快速上手 AI 编程

## 什么是 Vibe Coding？

Vibe Coding = **AI 氛围编程**，通过自然语言与 AI 对话，让 AI 生成代码，你只需测试、反馈、迭代。

**核心理念：**
- 用自然语言描述需求
- AI 生成可运行代码
- 快速迭代反馈
- 专注于创意而非语法

## 第二步：你的第一个项目

### 创建待办事项应用

#### 提示词模板

```
创建一个待办事项应用，要求：

1. 功能：
   - 添加、编辑、删除待办事项
   - 标记完成状态
   - 按日期排序

2. 技术栈：
   - Python Flask + HTML/JavaScript
   - 数据存储在 JSON 文件

3. 设计：
   - 简洁的用户界面
   - 响应式设计
   - 深色模式支持

4. 遵循 vibe-coding 中的 Python 规范
```

#### 迭代流程

```
第 1 轮：生成基础框架
你：运行代码，测试基本功能

第 2 轮：添加样式
你："界面太简单了，添加更多样式"
AI：[生成改进的 CSS]

第 3 轮：完善功能
你："添加编辑和删除功能"
AI：[生成完整 CRUD]

第 4 轮：测试和修复
你：运行测试，报告 bug
AI：修复问题
```

### Python 项目检查清单

- [ ] 遵循 PEP 8 风格
- [ ] 使用有意义的命名
- [ ] 函数不超过 20 行
- [ ] 每个函数只做一件事

## 第五步：代码审查

### 自查清单

运行代码后，检查：

```
1. 功能完整性
   [ ] 所有需求都实现了吗？
   [ ] 边界情况处理了吗？

2. 代码质量
   [ ] 有重复代码吗？
   [ ] 函数太长吗？
   [ ] 命名清晰吗？

3. 错误处理
   [ ] 异常捕获了吗？
   [ ] 有日志记录吗？
   [ ] 用户友好的错误提示？

4. 安全性
   [ ] 输入验证了吗？
   [ ] 密码加密了吗？
   [ ] SQL 注入防护？
```

## 常见问题

### Q1: AI 生成的代码有 bug 怎么办？

**A:** 这是正常的。解决方法：
- 复制错误信息给 AI
- 描述期望行为
- 让 AI 修复并重新生成

### Q2: AI 忘记了之前的代码怎么办？

**A:** 重新提供上下文：
- "这是我们之前讨论的登录功能，现在添加记住我选项"
- 用代码注释记录重要决策

### Q3: 如何提高代码质量？

**A:** 
- 明确技术要求
- 引用编码规范
- 多轮迭代优化
- 人工审查关键逻辑
