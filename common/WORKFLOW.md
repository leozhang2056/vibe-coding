# Vibe Coding 核心工作流

> 基于 Andrej Karpathy 的 Vibe Coding 理念

## 核心理念

Vibe Coding = **AI 协作编程**，完全沉浸在 AI 的氛围中，通过自然语言对话实现快速开发。

**关键特征：**
- 自然交互（用自然语言描述需求）
- 快速迭代（AI 生成代码 → 测试 → 反馈）
- 心流状态（专注于创意和逻辑，而非语法细节）

---

## 工作流程

### 1. 自然语言描述

用清晰、具体的自然语言描述需求：

```
❌ 糟糕的描述：
"做一个网站"

✅ 好的描述：
"使用 Python Flask 创建一个用户注册 API，要求：
1. 使用 JWT 认证
2. 密码使用 bcrypt 加密
3. 包含输入验证
4. 返回标准 JSON 响应
5. 遵循 SOLID 原则"
```

---

### 2. AI 代码生成

AI 分析需求并生成代码：
- 需求解析
- 代码生成
- 遵循最佳实践

---

### 3. 执行与观察

运行 AI 生成的代码：
- 代码执行
- 功能测试
- 问题识别

---

### 4. 反馈与修正

根据测试结果提供反馈：
- 错误报告
- 功能调整
- 优化建议

```
示例反馈：
"密码验证需要包含特殊字符要求"
"按钮颜色应该是蓝色"
"添加键盘支持"
```

---

### 5. 迭代循环

重复以上过程直到满足需求：
- 持续改进
- 增量开发
- 质量保证

---

## 项目分解策略

### 大项目分解

将大型项目分解为可管理的模块：

1. **用户界面层**
   ```
   "创建应用的主要布局，包含导航栏、侧边栏和主内容区域"
   ```

2. **数据层**
   ```
   "设计用户数据模型，包括注册、登录和个人资料管理"
   ```

3. **业务逻辑层**
   ```
   "实现核心功能逻辑，如搜索、过滤和数据处理"
   ```

## 多轮对话管理

### 保持上下文连续性

**上下文提示：**
```
"基于之前创建的用户认证系统，现在添加密码重置功能"
```

**引用之前的代码：**
```
"修改我们之前讨论的登录组件，添加'记住我'选项"
```

---

## 提示工程最佳实践

### 有效提示的特征

- **具体明确**：避免模糊的表述
- **技术栈说明**：明确指定使用的技术
- **约束条件**：说明性能、兼容性限制
- **预期结果**：描述期望的最终效果

### 示例

```
❌ 糟糕的提示：
"做一个登录功能"

✅ 好的提示：
"使用 Python Flask 和 JWT 创建一个用户登录功能，
包括：
1. 用户名和密码验证
2. 密码使用 bcrypt 加密
3. 登录成功返回 JWT token
4. 遵循 Clean Architecture 原则
5. 包含输入验证和错误处理"
```

## 常见问题与解决方案

### 1. AI 生成代码质量差

**解决方案：**
- 提供更具体的技术要求
- 分步骤生成，逐步完善
- 结合人工审查

### 2. 上下文丢失

**解决方案：**
- 在新对话中重新提供关键信息
- 使用代码注释记录设计决策
- 保持对话的连续性

### 3. 过度依赖 AI

**解决方案：**
- 平衡 AI 使用和传统学习
- 定期进行技术培训
- 建立代码审查机制

## 迭代式开发原则

### 小步快跑

- 每次只要求实现一个小功能
- 逐步完善
- 保持上下文

### 示例

```
第 1 轮：实现基本 CRUD
第 2 轮：添加数据验证
第 3 轮：添加缓存
第 4 轮：添加日志
第 5 轮：添加测试
```

---

## AI 编码检查清单

编码时，确保符合以下检查：

- [ ] 提示词清晰具体
- [ ] 包含技术栈说明
- [ ] 有明确的约束条件
- [ ] 描述了预期结果
- [ ] 分步骤迭代开发
- [ ] 保持上下文连续
- [ ] 每轮都测试反馈
